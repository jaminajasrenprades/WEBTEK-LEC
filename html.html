    • Hypertext Markup Language (HTML)

    A markup language (not a programming language) that is used to create web pages in the World Wide Web.

    It was introduced by the rules of Standard Generalized Markup Language (SGML) in document type definition (DTD), by Tim-Berners-Lee in the year 1991.

    Creates structured documents by denoting semantics for text like headings, lists, links, quotes etc.

    HTML Versions

    Version Description
    HTML 1.0 HTML 1.0 was the first release of HTML to the world. Not many people were involved in website creation at the time, and the language was very limiting. There really wasn’t much you could do with it bar getting some simple text onto the web. But then, just that got the beardos a-foamin’ back in the day.
    HTML 2.0 HTML 2.0 included everything from the original 1.0 specifications but added a few new features to the mix. HTML 2.0 was the standard for website design until January 1997 and defined many core HTML features for the first time.
    HTML 3.0 It included many new and improved abilities for HTML, and promised far more powerful opportunities for webmasters to design their pages. Sadly, the browsers were awfully slow in implementing any of the new improvements, only adding in a few and leaving out the rest. Partly, this failure can be attributed to the size of the overhaul and so the HTML 3.0 spec was abandoned.
    HTML 3.2 This was a toned-down change to the existing standards, leaving many of the big steps forward for later versions. Most of the extensions tags that had been introduced by Netscape (and to a lesser-extent, Microsoft) did not make it into these new standards. It soon caught on and became the official standard in January ’97, and today practically all browsers support it fully.

    HTML 4.01 This version is strict, transitional and uses frameset.

    HTML5 Elements

    header
    This element contains the header content of the site.
    footer
    This element contains the footer of the site.
    nav
    This element contains the navigation menu, or the other navigation functionality of the page.
    article
    This element is somewhat related to section tag. But it contains related individual standalone pieces of content. Ex: blog posts, videos, images, etc.
    section
    This element it used to group different articles.
    aside
    This element defines a block of content that is related to the main content around it, but not central to the flow of it.

    HTML VS. XHTML

    HTML and XHTML are quite related to each other but here are their possible differences.

    HTML XHTML
    Start or end tags are not required for every element. All elements must have a start tag.
    Tags and attributes are case-insensitive. Elements and attributes are case sensitive, it is mostly in lowercase.
    Attribues do not need to be quoted. Attribute values must be enclosed in quotes.
    The document must include an HTML5 DOCTYPE. XHTML documents must be well-formed.
    All the content can be put under the body element. All elements are nested properly.
